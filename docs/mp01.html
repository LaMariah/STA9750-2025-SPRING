Policy Analysis: NYC Payroll Optimization Strategies

A White Paper for the Commission to Analyze Taxpayer Spending (CATS)

Author

La Maria

Published

February 26, 2025

1 Executive Summary

This white paper presents an analysis of New York City’s payroll data with recommendations to optimize taxpayer spending. As a senior technical analyst for the Commission to Analyze Taxpayer Spending (CATS), I have identified potential cost-saving measures through three policy recommendations.

1.1 Quick Facts About NYC Payroll

  • The job title with the highest base rate of pay is Chief Medical Examiner
  • NYPD has the highest average total annual payroll per employee
  • NYPD also has the most employees on payroll each year
  • Fire Department has the highest overtime usage compared to regular hours
  • The city’s aggregate payroll has grown significantly over the past 10 years

2 Introduction

The Commission to Analyze Taxpayer Spending (CATS) aims to understand New York City’s expenses and identify opportunities for more effective use of taxpayer funds. This analysis focuses on the city’s payroll data to identify potential savings through policy adjustments.

3 Data Acquisition and Preparation

3.1 Data Source

Show code 
# Task 1: Data Acquisition
if(!file.exists("data/mp01/nyc_payroll_export.csv")){
  dir.create("data/mp01", showWarnings=FALSE, recursive=TRUE)
  
  ENDPOINT <- "https://data.cityofnewyork.us/resource/k397-673e.json"
  
  if(!require("httr2")) install.packages("httr2")
  library(httr2)
  
  if(!require("jsonlite")) install.packages("jsonlite")
  library(jsonlite)
  
  if(!require("dplyr")) install.packages("dplyr")
  library(dplyr)
  
  if(!require("readr")) install.packages("readr")
  library(readr)
  
  BATCH_SIZE <- 50000
  OFFSET     <- 0
  END_OF_EXPORT <- FALSE
  ALL_DATA <- list()
  
  while(!END_OF_EXPORT){
    cat("Requesting items", OFFSET, "to", BATCH_SIZE + OFFSET, "\n")
    
    req <- request(ENDPOINT) |>
      req_url_query(`$limit`  = BATCH_SIZE, 
                    `$offset` = OFFSET)
    
    resp <- req_perform(req)
    
    batch_data <- fromJSON(resp_body_string(resp))
    
    ALL_DATA <- c(ALL_DATA, list(batch_data))
    
    if(NROW(batch_data) != BATCH_SIZE){
      END_OF_EXPORT <- TRUE
      
      cat("End of Data Export Reached\n")
    } else {
      OFFSET <- OFFSET + BATCH_SIZE
    }
  }
  
  ALL_DATA <- bind_rows(ALL_DATA)
  
  cat("Data export complete:", NROW(ALL_DATA), "rows and", NCOL(ALL_DATA), "columns.")
  
  write_csv(ALL_DATA, "data/mp01/nyc_payroll_export.csv")
}

payroll_growth <- payroll_data %>%

if(nrow(payroll_growth) >= 2) {
  earliest_year <- min(payroll_growth$fiscal_year)
  latest_year <- max(payroll_growth$fiscal_year)
  earliest_payroll <- payroll_growth$total_payroll[payroll_growth$fiscal_year == earliest_year]
  latest_payroll <- payroll_growth$total_payroll[payroll_growth$fiscal_year == latest_year]

    geom_line(color = "blue", size = 1.2) +

6 Policy Analysis

6.1 Policy I: Capping Salaries at Mayoral Level

Show code 
# Find mayoral salary for each fiscal year
mayoral_salaries <- payroll_data %>%
  filter(title_description == "Mayor") %>%
  group_by(fiscal_year) %>%
  summarize(mayoral_salary = max(base_salary, na.rm = TRUE), .groups = "drop")

# For simplicity, use the maximum mayoral salary across all years
mayoral_cap <- max(mayoral_salaries$mayoral_salary, na.rm = TRUE)

# Calculate potential savings from implementing the cap
salary_cap_analysis <- payroll_data %>%
  mutate(
    over_cap = total_compensation > mayoral_cap,
    savings = if_else(over_cap, total_compensation - mayoral_cap, 0)
  ) %>%
  summarize(
    employees_affected = sum(over_cap, na.rm = TRUE),
    total_savings = sum(savings, na.rm = TRUE),
    percent_of_workforce = employees_affected / n() * 100
  )

# Display the results
salary_cap_analysis %>%
  mutate(
    total_savings = dollar(total_savings),
    percent_of_workforce = scales::percent(percent_of_workforce/100, accuracy = 0.01)
  ) %>%
  knitr::kable(caption = "Potential Impact of Salary Cap at Mayoral Level")

Many governments implement policies that cap the salaries of public employees at the level of the chief executive. This analysis examines the potential impact of capping NYC employee compensation at the mayoral salary level.

Based on our analysis, implementing a salary cap at the mayoral level would affect a small percentage of employees but could generate significant savings. However, careful consideration must be given to potential impacts on recruitment and retention of specialized talent.

6.2 Policy II: Increasing Staffing to Reduce Overtime Expenses

Show code 
# Calculate overtime analysis by agency and job title
overtime_analysis <- payroll_data %>%
  filter(!is.na(hourly_rate), !is.na(ot_hours), ot_hours > 0) %>%
  mutate(
    overtime_cost = ot_hours * hourly_rate * 1.5,
    regular_cost_equivalent = ot_hours * hourly_rate,
    potential_savings = overtime_cost - regular_cost_equivalent,
    full_time_equivalents_needed = ot_hours / 2000  # Assuming 2000 hours per year
  ) %>%
  group_by(agency_name, title_description) %>%
  summarize(
    total_ot_hours = sum(ot_hours, na.rm = TRUE),
    total_overtime_cost = sum(overtime_cost, na.rm = TRUE),
    total_regular_cost_equivalent = sum(regular_cost_equivalent, na.rm = TRUE),
    total_potential_savings = sum(potential_savings, na.rm = TRUE),
    total_fte_needed = sum(full_time_equivalents_needed, na.rm = TRUE),
    .groups = "drop"
  ) %>%
  filter(total_ot_hours > 1000) %>%  # Focus on significant overtime usage
  arrange(desc(total_potential_savings))

# Display top opportunities
overtime_analysis %>%
  slice_head(n = 10) %>%
  select(
    agency_name, 
    title_description, 
    total_ot_hours, 
    total_potential_savings,
    total_fte_needed
  ) %>%
  mutate(
    total_potential_savings = dollar(total_potential_savings),
    total_fte_needed = round(total_fte_needed, 1)
  ) %>%
  knitr::kable(caption = "Top 10 Opportunities for Overtime Reduction")

Overtime expenses significantly impact the city’s budget due to the 1.5x premium on overtime hours. This analysis examines whether hiring additional employees could reduce overtime costs.

Our analysis identifies significant potential savings from strategic hiring to replace overtime hours with regular-time work. Key departments like NYPD and Fire Department would benefit most from this approach.

6.3 Policy III: Implementing Flexible Work Arrangements

Show code 
# Analyze potential savings from capping overtime at 20 hours per employee
flexible_work_analysis <- payroll_data %>%
  filter(!is.na(hourly_rate), !is.na(ot_hours)) %>%
  mutate(
    current_ot_cost = ot_hours * hourly_rate * 1.5,
    capped_ot_hours = pmin(ot_hours, 20),  # Cap at 20 hours
    capped_ot_cost = capped_ot_hours * hourly_rate * 1.5,
    savings = current_ot_cost - capped_ot_cost
  ) %>%
  summarize(
    total_current_ot_cost = sum(current_ot_cost, na.rm = TRUE),
    total_capped_ot_cost = sum(capped_ot_cost, na.rm = TRUE),
    total_savings = sum(savings, na.rm = TRUE),
    employees_affected = sum(ot_hours > 20, na.rm = TRUE),
    percent_workforce_affected = employees_affected / n() * 100
  )

# Display results
flexible_work_analysis %>%
  mutate(
    total_current_ot_cost = dollar(total_current_ot_cost),
    total_capped_ot_cost = dollar(total_capped_ot_cost),
    total_savings = dollar(total_savings),
    percent_workforce_affected = scales::percent(percent_workforce_affected/100, accuracy = 0.01)
  ) %>%
  knitr::kable(caption = "Potential Impact of Flexible Work Policy with Overtime Cap")

As a third policy option, implementing flexible work arrangements with caps on overtime hours could provide both financial savings and improved work-life balance for city employees.

Our analysis shows that implementing a flexible work arrangement policy with overtime caps would generate significant savings while affecting only a portion of the workforce. This policy could also improve employee satisfaction and productivity.

7 Conclusion

Based on our comprehensive analysis of NYC payroll data, we recommend a multi-faceted approach to optimize taxpayer spending:

  1. Primary Strategy: Implement Policy II (Strategic Hiring to Reduce Overtime) for the highest potential savings with minimal disruption.

  2. Secondary Strategy: Phase in Policy III (Flexible Work Arrangements) to address work-life balance while generating additional savings.

  3. Long-term Consideration: Evaluate Policy I (Mayoral Salary Cap) on a targeted basis for positions where public sector compensation significantly exceeds private sector benchmarks.

7.1 Limitations and Future Research

This analysis has several limitations:

  • Data quality issues, such as missing values and inconsistent coding
  • Inability to account for all factors affecting hiring decisions
  • Limited information on job market conditions for specialized roles

Future research could: - Compare NYC salaries with those in other major cities - Analyze the impact of collective bargaining agreements on overtime usage - Evaluate the effectiveness of flexible work arrangements in public sector settings
